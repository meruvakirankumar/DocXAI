DocXAI

AI-Powered Document & Test Automation Engine
GE Vernova — GCA Hackathon 2026  |  Team 19 — MFG  |  .NET 10 • Vertex AI (Gemini 2.0 Flash) • Google Cloud Platform

1. Problem Statement

Converting design documents into functional specifications and test cases is manual, slow, and error-prone. Business analysts write specs by hand; QA engineers separately author test scripts — often weeks apart. Key pain points:

  • Slow turnaround: Specs take days to draft; test scripts take additional days, delaying the entire SDLC.
  • Inconsistency: Manual translation introduces gaps, ambiguities, and missed requirements.
  • No traceability: No automated link between design → spec → tests.
  • Version friction: Iterations managed in shared drives or email — files get overwritten or lost.
  • Repetitive effort: Teams repeat the same read → write → test pattern with no reuse or automation.

2. Use Case

A project team receives a design document (.docx or Markdown). They need to:

  1. Produce a comprehensive Functional Specification capturing all requirements and edge cases.
  2. Generate an executable Playwright test suite (TypeScript) that validates the system against the spec.
  3. Store all artifacts with automatic versioning — nothing is ever overwritten.
Key Insight: The functional spec and test cases are two views of the same requirements. Generating both from a single document via AI ensures 100% traceability and eliminates the "telephone game" between design, spec, and QA.

3. Proposed Solution

DocXAI automates the design → spec → test pipeline using Gemini 2.0 Flash. Built as a clean-architecture .NET 10 app on Cloud Run with a browser UI for human-in-the-loop control.

Core Workflow

  1. Upload — drag a design doc into the browser, provide a Solution Name.
  2. AI Spec Generation — Gemini produces a Functional Specification (Markdown).
  3. Review & Save — spec displayed in a Markdown viewer; copy or download as .docx.
  4. On-Demand Test Gen — user clicks "Generate Test Cases"; Gemini produces Playwright TypeScript.
  5. Review Tests — syntax-highlighted viewer with download and copy.

Key Design Decisions

  • Human-in-the-loop: test generation is decoupled — user reviews spec first.
  • Auto versioning: _v1, _v2, _v3… — never overwrites.
  • Organised storage: {Sol}/functional/ for specs, {Sol}/test cases/ for tests.
  • Zero-framework frontend: vanilla JS — no dependencies, instant load.

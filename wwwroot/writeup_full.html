DocXAI

AI-Powered Document & Test Automation Engine

1. Problem Statement

In large-scale enterprise software development, converting design documents into structured functional specifications and executable test cases is a manual, time-consuming, and error-prone process. Business analysts write specifications by hand, and QA engineers separately author test scripts вҖ” often weeks apart. This creates several pain points:

  • Slow turnaround: Functional specs take days to draft; test scripts take additional days, delaying the entire SDLC.
  • Inconsistency: Manual translation from design to spec to tests introduces gaps, ambiguities, and missed requirements.
  • No traceability: There is no automated link between the original design document, the generated spec, and the resulting test suite.
  • Version control friction: Multiple iterations of specs and tests are managed in shared drives or email, leading to overwritten files and lost history.
  • Repetitive effort: Teams across projects repeat the same pattern вҖ” read design, write spec, write tests вҖ” with no reuse or automation.

2. Use Case

A project team receives a design document (e.g., a UI wireframe specification, a workflow description, or an API contract in .docx or Markdown format). They need to:

  1. Produce a comprehensive Functional Specification that captures all requirements, acceptance criteria, and edge cases.
  2. Generate an executable Playwright test suite (TypeScript) that validates the system against the specification.
  3. Store all artifacts with automatic versioning so nothing is ever overwritten.
Key Insight: The functional spec and test cases are two views of the same requirements. By generating both from a single design document using AI, we ensure 100 % traceability and eliminate the "telephone game" between design, spec, and QA.

3. Proposed Solution

DocXAI is a web-based platform that automates the entire design вҶ’ spec вҶ’ test pipeline using Google's Gemini 2.0 Flash large language model. The solution is built as a clean-architecture .NET 10 application deployed on Google Cloud Run, with a browser-based UI for human-in-the-loop control.

Core Workflow

  1. Upload: The user drags a design document into the browser and provides a Solution Name.
  2. AI Spec Generation: The document is sent to Vertex AI (Gemini) which produces a detailed Functional Specification in Markdown.
  3. Review & Save: The spec is displayed in a rich Markdown viewer. The user can review, copy, or download it as a Word (.docx) file.
  4. On-Demand Test Generation: When satisfied with the spec, the user clicks "Generate Test Cases". Gemini produces a complete Playwright TypeScript test suite.
  5. Review Tests: The test file is displayed in a syntax-highlighted TypeScript viewer with download and copy options.

Key Design Decisions

  • Human-in-the-loop: Test generation is intentionally decoupled from spec generation вҖ” the user reviews the spec before creating tests.
  • Automatic versioning: All artifacts use {SolutionName}_functional_v1.docx, v2, v3вҖҰ naming. The system never overwrites previous outputs.
  • Organised storage: Each solution gets its own GCS folder with sub-folders: functional/ for specs and test cases/ for test scripts.
  • No vendor lock-in on the frontend: The UI is vanilla JavaScript вҖ” no framework dependencies, instant load times.
